# Lab Exercise: Creating Digital Twins with Gazebo and Isaac Sim

## Objective
In this lab, you'll create a digital twin of a simple humanoid robot in both Gazebo and Isaac Sim, comparing the capabilities of each simulation environment.

## Prerequisites
- Gazebo Classic or Garden installed
- Isaac Sim (NVIDIA Omniverse) installed (optional)
- Basic understanding of URDF/SDF robot models

## Theory
Digital twins enable rapid prototyping, testing, and AI training without risk to physical hardware. This lab compares Gazebo and Isaac Sim for humanoid robot simulation.

## Implementation

### Step 2: Setting up Gazebo Simulation
To run this model in Gazebo, create a launch file:

```xml
<!-- launch/humanoid_gazebo.launch -->
<launch>
  <!-- Load the robot description -->
  <param name="robot_description" command="$(find executable which xacro) $(find package name)/urdf/humanoid.urdf.xacro" />
  
  <!-- Spawn the robot in Gazebo -->
  <node name="spawn_urdf" pkg="gazebo_ros" type="spawn_entity.py" args="-entity simple_humanoid -database robot_description" />
  
  <!-- Launch Gazebo with a world file -->
  <include file="$(find gazebo_ros)/launch/empty_world.launch">
    <arg name="world_name" value="$(find package name)/worlds/simple_room.world" />
  </include>
</launch>
```

And then run:
```bash
# Launch the simulation
roslaunch package_name humanoid_gazebo.launch

# In another terminal, you can control the robot with commands like:
rostopic pub /left_shoulder_position_controller/command std_msgs/Float64 "data: 0.5"
```

### Step 3: Isaac Sim (Conceptual Implementation)
For Isaac Sim, you would typically create the robot using Omniverse's UI or Python API:

```python
# Isaac Sim robot creation (conceptual)
import omni
from omni.isaac.core import World
from omni.isaac.core.utils.stage import add_reference_to_stage
from omni.isaac.core.utils.prims import create_primitive, get_prim_at_path
from pxr import Gf

# Create a world
world = World(stage_units_in_meters=1.0)

# Add your robot from USD file (this would be your humanoid model converted to USD)
add_reference_to_stage(
    usd_path="path/to/humanoid.usd", 
    prim_path="/World/HumanoidRobot"
)

# Add objects to interact with
create_primitive(
    prim_path="/World/Cube",
    primitive_type="Cube",
    position=Gf.Vec3f(0.5, 0.0, 0.5),
    scale=Gf.Vec3f(0.1, 0.1, 0.1)
)

# Reset and step the world
world.reset()
for i in range(100):
    world.step(render=True)
    # Control the robot here with joint commands
```

### Step 4: Physics Comparison
Compare the physics simulation between environments:

```python
# Compare physics properties
def compare_physics():
    print("Gazebo Physics:")
    print("- Uses ODE, Bullet, or DART engines")
    print("- Good for basic dynamics and collision detection")
    print("- Moderate rendering quality")
    
    print("\nIsaac Sim Physics:")
    print("- Uses PhysX engine")
    print("- Accurate contact simulation")
    print("- Photorealistic rendering")
    print("- Synthetic data generation capabilities")

compare_physics()
```
